# VOVAN Site Bridge Boundary

## Название bridge

Механизм связи сайта с ВОВАНОМ называется `vovan_cli_bridge` или `vovan_site_bridge`.

Bridge — это интеграционная граница между VLADCHER_ru и локальным VOVAN-CLI. Он не переносит локальное вычисление в сайт.

## Роли

VLADCHER_ru не должен сам выполнять локальный OCR и не должен требовать входящих соединений на локальную машину оператора.

Распределение ответственности:

- сайт создаёт job;
- VOVAN-CLI забирает job в pull-mode;
- VOVAN-CLI скачивает input;
- VOVAN-CLI обрабатывает input локально;
- VOVAN-CLI отправляет complete/fail result.

Такой flow оставляет приватные вычисления на локальной машине и не требует открывать локальный HTTP server наружу.

## Pull-mode

Pull-mode означает, что VOVAN-CLI сам инициирует исходящие запросы к VLADCHER_ru:

- получает следующую job;
- скачивает входной файл по разрешённому endpoint;
- отправляет успешный результат;
- отправляет controlled failure, если обработка невозможна.

Входящие соединения на локальную машину не нужны.

## Безопасность данных

Raw приватные документы, сканы, PDF, OCR samples и private diagnostics не должны попадать в GitHub. В репозитории допустимы только безопасные fixtures, generated samples или документация без приватного содержимого.

Bridge не должен печатать токены, секреты, auth headers, cookies или реальные env-значения. Логи должны быть короткими, контролируемыми и пригодными для публикации без раскрытия приватных файлов, локальных абсолютных путей или sensitive diagnostics.
